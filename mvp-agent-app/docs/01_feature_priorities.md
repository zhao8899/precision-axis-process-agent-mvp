# Docker化本地MVP控制台功能优先级

本文面向 `mvp-agent-app` 实现团队，用于把 `MVP开发交付包/` 转化为本地可部署、可测试、可交接的最小控制台。功能边界必须追溯到交付包，不新增资料包外能力。

## 一句话边界

智能体负责“提示、比对、归档辅助”，学生负责“操作、判断、放行、处置”。

## P0 必须实现

| 功能 | 实现目标 | 来源依据 | 不可越界 |
|---|---|---|---|
| 交付包资料加载 | 能读取并展示系统提示词、KB-01至KB-04、合并知识库、T01-T08测试口径 | `README.md`、`10_平台导入顺序与操作手册.md` | 不改写原始知识库含义 |
| 工艺辅助问答 | 支持围绕工艺问答、参数建议、G1-G8、异常处置、数据归档、现场话术发问 | `00_MVP范围说明.md`、`01_系统提示词_正式版.md` | 不泛化成普通机械问答 |
| 固定回答边界 | 请求工艺参数、质量控制点、异常处理、应用价值、现场话术时，引导模型采用交付包定义格式 | `01_系统提示词_正式版.md` | 不给唯一正确参数，不自动放行 |
| T01-T08测试执行 | 控制台可逐条运行T01-T08，保存真实模型回答摘要、人工判定和备注 | `05_KB-04_测试用例与验收口径.md`、`11_T01-T08标准判定稿.md` | 不用预写答案替代真实模型测试 |
| 验收记录导出 | 输出可回填到测试验收记录的结构化结果 | `08_测试验收记录表.md` | 不伪造通过结论 |
| 合规拦截提醒 | 对自主开发AI、自动判断合格、企业真实订单、涉密资料、跳过首检等表达给出风险提示 | `01_系统提示词_正式版.md`、`15_需求追踪矩阵.md` | 不放宽交付包边界 |
| Docker本地运行 | 提供本地容器启动后的控制台访问入口和基础健康检查 | `14_开发流程与任务分解.md` | 不把容器启动等同于业务验收 |

## P1 建议实现

| 功能 | 实现目标 | 来源依据 |
|---|---|---|
| G1-G8规则浏览 | 按编号查看检测对象、标准/数据依据、不合格处置、建议输出物和禁止表述 | `03_KB-02_G1-G8质量控制点清单.md` |
| 异常处置模板 | 按四类异常生成“五步法”记录草稿 | `04_KB-03_异常处置规则.md` |
| 演示问题模式 | 固定展示客户演示脚本问题，便于演示前复核 | `06_客户演示脚本.md` |
| 资料脱敏检查 | 展示脱敏检查项并记录是否已确认 | `09_资料脱敏检查表.md` |
| 版本与配置记录 | 记录本地模型供应商、知识库版本、测试时间和测试人 | `07_平台配置记录表.md`、`12_MVP版本记录.md` |

## P2 暂缓

| 功能 | 暂缓原因 |
|---|---|
| 多用户权限后台 | 明确不纳入MVP范围 |
| 机床、检测设备、数控系统接口 | 明确不纳入MVP范围 |
| 自动读取企业完整图纸 | 明确不纳入MVP范围，且存在涉密风险 |
| 生产级审计系统 | 明确不纳入MVP范围 |
| 新控制点、新工艺、新岗位流程 | 无法追溯到当前资料包 |

## 页面优先级

1. 首页：显示项目定位、严格边界、当前知识库版本、Docker健康状态。
2. 问答控制台：输入问题、选择场景、查看模型回答、查看边界提醒。
3. 知识库浏览：只读展示交付包导入内容和G1-G8规则。
4. T01-T08测试：逐条运行、记录真实回答、人工判定。
5. 验收与导出：汇总通过状态、失败项、复测建议和可交接记录。

## 成功标准

- 本地Docker可启动并访问控制台。
- 控制台能基于交付包内容完成真实问答。
- T01-T08可执行、可记录、可人工判定。
- 所有回答和记录都保持“不夸大、不虚构、不替代学生放行”的底线。
