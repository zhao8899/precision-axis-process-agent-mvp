# 精轴智控工艺辅助智能体 MVP开发交付包

本目录是“精轴智控工艺辅助智能体（基础版）”MVP真实开发第一版资产，不是一次性演示稿。

## 开发定位

MVP基于学校授权AI平台/大模型工具配置，目标是形成可真实问答、可测试、可客户演示、可继续迭代的基础版智能体。

一句话边界：

> 智能体负责“提示、比对、归档辅助”，学生负责“操作、判断、放行、处置”。

## 使用顺序

1. 先阅读 `00_MVP范围说明.md`，确认客户认可MVP边界。
2. 按 `09_资料脱敏检查表.md` 完成资料审核。
3. 将 `02` 到 `05` 作为知识库文件导入学校授权AI平台。
4. 将 `01_系统提示词_正式版.md` 配置为系统提示词。
5. 按 `07_平台配置记录表.md` 记录真实配置。
6. 按 `10_平台导入顺序与操作手册.md` 完成导入后首轮检查。
7. 按 `08_测试验收记录表.md` 跑T01-T08真实测试，并用 `11_T01-T08标准判定稿.md` 判定是否通过。
8. 若测试失败，按 `16_缺陷修正与迭代规则.md` 修正并复测。
9. 测试通过后，按 `06_客户演示脚本.md` 做客户演示。
10. 全程用 `12_MVP版本记录.md`、`13_下一步执行清单.md` 和 `15_需求追踪矩阵.md` 记录状态。

## 开发管理

- `14_开发流程与任务分解.md` 是当前开发主流程。
- `15_需求追踪矩阵.md` 是防止范围偏离的依据。
- `16_缺陷修正与迭代规则.md` 是真实测试后唯一允许的修正流程。
- `17_平台导入_合并知识库.md` 是平台能力不足时的兼容导入方案。

## 严格边界

- 不声称自主开发AI系统。
- 不自动判断合格或自动放行。
- 不编造实测值、企业订单、企业利润、客户信息。
- 不上传企业完整未脱密图纸、合同、订单价格、学生隐私。
- 训练测算值必须标注来源，正式材料必须替换为学校实测值或企业确认值。
