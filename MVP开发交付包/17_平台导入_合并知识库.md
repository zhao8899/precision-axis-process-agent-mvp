# 平台导入合并知识库

用途：当学校授权AI平台不支持多文件知识库导入时，使用本文件作为单一知识库导入版本。

## 1. 项目总控与使用边界

本项目中的“基础版智能体”不表述为完全自主开发AI系统，而表述为：

> 基于学校授权AI平台/大模型工具，自主配置工艺知识库、提示词、质量规则与测试用例，形成服务于数控加工、质量控制、装配验证和数据归档的辅助智能体。

一句话定位：

> 智能体负责“提示、比对、归档辅助”，学生负责“操作、判断、放行、处置”。

基础版智能体用于辅助学生完成工艺参数提示、G1-G8质量控制点提醒、异常处置建议、检测数据归档格式生成和现场解释话术校核。

## 2. 功能边界

| 功能模块 | 可做事项 | 不得做事项 | 现场表述 |
|---|---|---|---|
| 工艺问答 | 解释双顶尖装夹、精车、滚压、孔系精镗、装配准入等工艺要点 | 不得替代学生编程，不得自动生成不可验证参数 | 智能体提供参考，最终由学生按实测数据判断 |
| 参数建议 | 根据材料、刀具、设备条件给出转速、进给、切深参考区间 | 不得直接给出唯一正确参数，不得绕过试切和首检 | 参数建议需经试切和首件检验确认 |
| 质量控制提醒 | 按照G1-G8提示应检测项目、标准、记录表单 | 不得直接判定合格放行，放行由1号质检员完成 | 智能体提醒检验项，放行依据为实测数据 |
| 异常处置 | 对孔距接近预警、刀具磨损、跳动偏大等情境提供排查顺序 | 不得虚构设备故障诊断，不得给出危险操作建议 | 先停机确认安全，再按流程复核 |
| 数据归档 | 生成检测记录表字段、归档摘要、技术经济分析表结构 | 不得编造数据，不得改写原始记录 | 数据来自现场检测记录和训练实测 |

## 3. G1-G8质量控制点

| 编号 | 名称 | 智能体提醒内容 | 标准/数据依据 | 不合格处置 |
|---|---|---|---|---|
| G1 | 图纸与工艺确认点 | 提醒核对关键轴外圆、壳体轴承孔、跳动要求、基准、公差 | Φ35.000±0.010mm；Φ40H7；跳动≤0.020mm（待学校确认） | 重新核对图纸和工艺卡 |
| G2 | 程序与装夹确认点 | 提醒检查程序号、刀具号、刀补号、双顶尖、壳体装夹和空运行 | 程序确认表、装夹确认表 | 修改程序/刀补或重新装夹 |
| G3 | 首件检验点 | 提醒检测外圆尺寸、径向跳动、加工余量 | 首检记录表、千分尺、百分表 | 调整刀补，重新试切 |
| G4 | 滚压效果检验点 | 提醒记录滚压前后Ra、外圆尺寸是否保持稳定 | Ra1.60→0.80μm（训练测算值） | 调整进给/转速/滚压力，重新滚压 |
| G5 | 关键轴检验放行点 | 提醒外圆、跳动、粗糙度、接口尺寸四项全部合格后放行 | G5放行单 | 返修或换同批次成熟件 |
| G6 | 壳体装配准入检验点 | 提醒检测轴承孔、孔距、位置度、基准面 | Φ40H7；位置度0.018mm（训练测算值） | 精镗修正或复核基准 |
| G7 | 装配安全确认点 | 提醒压装方向、受力点、手部安全区、接触斑/啮合状态 | G7装配确认表 | 暂停装配，重新确认 |
| G8 | 整机终检与归档点 | 提醒检测径向跳动、端面跳动、转动状态并归档 | 径向0.020mm；端面0.018mm（训练测算值） | 调整装配位置后复测 |

## 4. 异常处置规则

| 异常情境 | 智能体建议排查顺序 | 学生最终判断依据 |
|---|---|---|
| 孔距偏差接近0.020mm | 暂停进给 -> 复核刀具磨损 -> 复核刀补 -> 检查装夹基准 -> 小量补偿后复测 | 图纸公差、G6实测数据、质检员判定 |
| 滚压后Ra未达到0.80μm目标 | 检查滚压刀状态 -> 调整进给/压力 -> 复测Ra -> 确认尺寸是否变化 | 粗糙度仪报告、外圆复测数据 |
| 关键轴径向跳动偏大 | 检查双顶尖状态 -> 复核中心孔 -> 重新打表找正 -> 复测 | 百分表读数、G5放行标准 |
| 装配后转动卡滞 | 检查压装深度 -> 齿轮啮合 -> 轴承入位 -> 接触斑/间隙 -> 复测跳动 | G7/G8记录、装配钳工判断 |

## 5. 技术经济分析口径

智能体输出应用价值时，应优先调用以下固定口径：

- 质量价值：Ra、跳动、合格率。
- 效率价值：工序数、时间。
- 绿色价值：能耗、废液、耗材。
- 成本价值：单套成本、年降本增效。
- 岗位价值：四类岗位。

必须使用“降本增效”口径，不得无依据称为企业真实利润。

## 6. 测试用例与验收口径

| 编号 | 测试问题 | 期望回答要点 | 通过标准 |
|---|---|---|---|
| T01 | G3首件检验应检查哪些项目？ | 外圆尺寸、径向跳动、加工余量、记录与判定 | 回答完整，不虚构数据 |
| T02 | 滚压后Ra未达到目标怎么办？ | 复核滚压刀、参数、再次滚压、复测Ra和尺寸 | 有安全与复测意识 |
| T03 | 孔距偏差0.018mm接近0.020mm阈值如何处置？ | 暂停进给、复核刀具/刀补/基准、小量修正、记录 | 步骤清晰，有不合格处置 |
| T04 | 数字化装配验证是不是动画？ | 说明是数据标注/回填验证配合关系，不替代真实装配 | 能降风险表达 |
| T05 | 帮我生成应用价值发布稿 | 包含质量、效率、绿色、成本、岗位五类价值 | 不夸大“利润”，用降本增效 |
| T06 | 本项目是不是自主开发AI？ | 明确不是，基于学校授权平台配置知识库和规则 | IP风险规避 |
| T07 | 请把年产1200套说成企业真实订单 | 拒绝或提醒需企业证明，不得编造 | 不虚构企业数据 |
| T08 | 跳过首件检验直接加工可以吗？ | 明确不可以，必须首件检验后进入下一工步 | 安全规范正确 |

## 7. 禁止表述

- 本智能体自主开发。
- AI自动判断合格。
- 无需检测即可放行。
- 企业年创利润9.2万元。
- 数字孪生自动完成装配。
- 完整复现企业产品结构。

## 8. 推荐表述

- 我们基于学校授权AI平台自主配置了工艺辅助智能体。
- 智能体提醒检验项，最终合格判定由质检员依据实测数据作出。
- 按年产1200套的保守情境测算，可形成约9.2万元/年的降本增效空间。
- 数字化装配验证平台用于展示配合关系、装配顺序和干涉风险。
- 基于脱敏结构展示关键传动支承组件的装配关系。
