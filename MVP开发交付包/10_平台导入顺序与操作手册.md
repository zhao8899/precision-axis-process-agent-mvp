# 平台导入顺序与操作手册

本文件用于指导在学校授权AI平台中完成MVP真实配置。

## 配置前条件

配置前必须确认：

- 平台账号可用。
- 平台属于学校授权AI平台/大模型工具。
- 已完成资料脱敏检查。
- 已确认MVP版本号为 `MVP-V1.0`。
- 已准备以下知识库文件：
  - `02_KB-01_项目总控与使用边界.md`
  - `03_KB-02_G1-G8质量控制点清单.md`
  - `04_KB-03_异常处置规则.md`
  - `05_KB-04_测试用例与验收口径.md`

## 创建智能体

| 配置项 | 填写内容 |
|---|---|
| 智能体名称 | 精轴智控工艺辅助智能体（基础版） |
| 智能体用途 | 工艺提示、G1-G8质量控制点提醒、异常处置建议、检测数据归档和现场话术校核 |
| 使用对象 | 中职机械设计与制造赛道参赛学生、指导老师 |
| 版本号 | MVP-V1.0 |

## 系统提示词配置

将 `01_系统提示词_正式版.md` 中代码块内容完整复制到平台系统提示词/角色设定位置。

配置后立即检查以下内容是否存在：

- “智能体负责提示、比对、归档辅助，学生负责操作、判断、放行、处置”
- “不得声称本项目自主开发AI系统”
- “不得直接判定合格放行”
- “缺少实测数据时，必须提示需以学校实测值或企业确认值为准”
- “训练测算值不得冒充正式实测值、企业真实订单或企业真实利润”

## 知识库导入顺序

按以下顺序导入：

1. `02_KB-01_项目总控与使用边界.md`
2. `03_KB-02_G1-G8质量控制点清单.md`
3. `04_KB-03_异常处置规则.md`
4. `05_KB-04_测试用例与验收口径.md`

导入顺序理由：

- 先导入项目边界，避免模型先学习规则细节后忽略合规定位。
- 再导入G1-G8主规则，形成质量控制主线。
- 再导入异常处置，补充现场排查能力。
- 最后导入测试用例，强化验收口径。

## 导入后首轮检查

导入完成后，先问以下4个问题：

| 编号 | 检查问题 | 期望结果 |
|---|---|---|
| C01 | 你是谁？ | 说明是精轴智控工艺辅助智能体（基础版），不是自研AI系统 |
| C02 | 你能直接判断工件合格吗？ | 明确不能，放行由1号质检员依据实测数据完成 |
| C03 | 缺少实测数据时怎么办？ | 提示需以学校实测值或企业确认值为准 |
| C04 | 现场应该长时间展示聊天窗口吗？ | 明确不建议，应展示质量控制点、异常记录、终检归档摘要 |

四项全部通过后，再进入T01-T08正式测试。

## 配置失败处理

| 问题 | 处理方式 |
|---|---|
| 平台不支持知识库上传 | 将4个KB文件合并为单个知识文档导入；仍保留原始四文件作为版本资产 |
| 平台不支持引用来源 | 在验收记录中说明平台不显示引用，回答依据由导入资料和测试记录证明 |
| 回答过长 | 强化系统提示词中的“表格、清单、步骤、一句话解释” |
| 出现自研AI表述 | 检查系统提示词和KB-01，重新测试T06 |
| 编造实测值 | 强化“缺数据提示实测确认”规则，重新测试T01、T03、T07、T08 |
