# MVP范围说明

## 目标

开发一个真实可运行的“精轴智控工艺辅助智能体（基础版）”，用于辅助“精轴智控·孔系协同”项目的训练、测试和客户演示。

MVP必须证明：

- 能围绕项目资料回答，不泛化成普通机械问答。
- 能围绕G1-G8质量控制点提供提醒。
- 能给出资料包内定义的异常处置顺序。
- 能生成检测记录字段、归档摘要和现场解释句。
- 能识别并拒绝越权、虚构、涉密和夸大表达。

## MVP包含范围

| 功能模块 | MVP能力 | 输出形态 |
|---|---|---|
| 工艺问答 | 解释双顶尖装夹、精车、滚压、孔系精镗、装配准入等工艺要点 | 清单、步骤、一句话解释 |
| 参数建议 | 根据材料、刀具、设备条件给转速、进给、切深参考区间 | 参考参数、确认依据、首检提醒、风险提醒 |
| 质量控制提醒 | 按G1-G8提示检测项目、标准、记录表单和不合格处置 | G1-G8检查清单、记录字段 |
| 异常处置 | 针对孔距预警、Ra未达标、跳动偏大、装配卡滞给排查顺序 | 五步法、异常处置记录 |
| 数据归档 | 生成检测记录字段、G8归档摘要、技术经济分析表结构 | 表格、摘要 |
| 现场话术 | 校核现场主动解释句，避免不合规表达 | 动作一句话、数据一句话、结论一句话 |

## 不纳入MVP范围

- 不开发自研AI模型。
- 不开发机床、检测设备或数控系统接口。
- 不自动读取企业完整图纸。
- 不自动生成不可验证参数。
- 不自动判断合格放行。
- 不做多用户权限后台。
- 不做生产级审计系统。
- 不扩展当前资料包之外的新控制点、新工艺、新岗位或新业务流程。

## 成功标准

MVP通过以下标准即具备客户演示和下一阶段开发基础：

- T01-T08真实模型测试全部通过。
- 回答不夸大、不虚构数据、不替代学生放行判断。
- 能围绕G1-G8提供清晰提醒。
- 能输出规范表格或清单。
- 能识别涉密信息并提醒脱敏。
- 能以中职学生可理解的语言解释关键工艺。

## 数据口径

资料包中出现的以下数据均需保留来源标注：

- `Φ35.000±0.010mm`
- `Φ40H7`
- `跳动≤0.020mm`
- `Ra1.60→0.80μm`
- `位置度0.018mm`
- `径向0.020mm`
- `端面0.018mm`
- `年产1200套`
- `单套成本220元降至158元`

凡标注为“训练测算值”的内容，正式材料必须替换为学校实测值或企业确认值。
